uNav

Indoor Navigation App for Visually Impaired Users.


OVERVIEW

BACKGROUND

One challenge blind people face is way-finding indoors. Current GPS solutions are not fine grained enough to accurately pinpoint detailed information about locations within a building. Some visually impaired users may not know what features and amenities a building even has and may thus never take advantage of them. A solution to this problem is an application that focuses on providing users with real-time, crowd-sourced indoor navigation of public spaces. uNav attempts to solve this design problem.

CONCEPT

The design approach in which we based many of our design details and functionality is social accessibility. Our goal was to create a design that was aesthetically appealing and acceptable for our target users – visually and non-visually impaired users. Our design consists of a welcome screen and a main navigation panel where users input the specific location they want to navigate to, using the built in microphone or keyboard. Upon selecting a route, the user will see a screen where one half of the screen is a floor plan with arrows guiding their path, and the other half with instructions. Our “Amenities” and “Detour” features allow users to help find features within a building and find their way back. By configuring the verbosity, our application can direct our user where to go based on either the simplest instructions, using a choice of text, voice, or both, or provide detailed information, such as room numbers, water fountains and restrooms as they are passed. Other features also include creating notes for specific locations, sound and color settings, favorites, accessibility modes, call for help and an offline mode feature.


DESIGN PROCESS

We began the design process brainstorming and sketching potential solutions for an indoor way finding application. Our designs were based on requirements from our expert users and social accessibility design approach. We combined functions and features of the best solutions and came up with our best solution. During this process we also defined our uses cases and personas. We then developed our design by paper prototyping and storyboarding. After multiple user testing sessions, we finally went on to create a high-fidelity prototype using Axure.

Big Time Brewery

A modified website for Seattle's Original Brewpub.

OVERVIEW

PROJECT DESCRIPTION

The aim of this project was to re-design and implement Big Time Brewery’s website. Ideally we planned to re-do the entire website but given our time constraints, we focused on the following pages: Homepage, The Brewers and The Grub. Big Time Brewery’s current website is not very aesthetically pleasing, and navigating between pages is inefficient. We re-designed their website in order to implement responsive web design with bootstrap and making it visually engaging and efficient. The key highlights of the new website are the use of HTML and CSS for the navigation bar and carousel of images in on the Homepage. We also incorporated several JavaScript elements in the “Beers” section where owner can list what is currently available on tap, and add new brews to this dataset. This would enable customers to see all the beers on tap a long with their description. Furthermore, JavaScript is also incorporated to allow customers to sign up and receive newsletters or updates from Big Time to enhance user experience.

PROGRAMMING LANGUAGES

HTML | CSS | JavaScript | Bootstrap

Memory Game

Memory Game Website.

OVERVIEW

PROJECT DESCRIPTION

The challenge for this project was to create a memory game. The requirements for this game were that it must start with a grid of 16 tiles turned faced down. Each card has a duplicate, so there are 8 matching pairs randomly distributed in the grid. The player turns over two tiles at a time trying to find matching pairs. If the two tiles match, they remain turned over. If they don’t both tiles are flipped back over and the player tries again. The game ends when all pairs have been matched. This game uses the jQuery library to catch events and manipulate the DOM. It also uses Lo-Dash library utility functions to allow random shuffling. Additionally, the game keeps track of elapsed time, number of matches found, remaining matches and matches missed.

PROGRAMMING LANGUAGES

HTML | CSS | JavaScript | jQuery | Lo-Dash | DOM

Employee Address Book

Address Book Webpage

OVERVIEW

PROJECT DESCRIPTION

This website is a directory of Employees with their picture, title, phone number and email addresses. The client can search for a particular Employee by their first name, last name, title, phone number or email address. This will populate the list of employees that match the search query. The client is also able to sort by Last Name, First Name and Department after a search has been requested. This website was built based on Angular properties; it merges data with a template using Angular. It involved creating an Angular module, and defining a controller in that module and initializing the controller’s scope with data and settings. It also demonstrates mastery in adding that application and controller to an HTML page and adding the various attributes necessary to repeat blocks of HTML for each record, along with making it include or omit elements depending on whether particular record properties are present. It also allows dynamically filtering and re-sorting the records based on user search and sort by actions.

PROGRAMMING LANGUAGES

HTML | CSS | JavaScript | Bootstrap | Angular JS

Orbit

Mobile Design Language

OVERVIEW

PROJECT DESCRIPTION

The premise of this assignment is to create a new mobile design language. This is an ongoing class project that is still currently being developed. We are working in teams to build any mobile design of our choice. I am working under the Application and System Management team to design how users are going to interact with the device.

The main goal of this project is to create guidelines on how the user will manage multiple apps and trigger device level functions. This includes detailed sketches and diagrams of gestures such as application switching, launching new applications, closing applications and application drawers. In addition to those features, we are also designing a method to alert and notify users of events that happen to applications outside of the current view. This includes how single and multiple alerts are visible and how to dismiss alerts. Other design features also include locking and unlocking the device.

DESIGN TOOLS

Sketching | Lucid Chart Wireframing | Trello
